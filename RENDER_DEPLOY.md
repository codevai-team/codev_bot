# Деплой на Render.com

## Шаги для деплоя:

### 1. Создайте аккаунт на Render.com
Перейдите на https://render.com и зарегистрируйтесь (можно через GitHub)

### 2. Создайте новый сервис
1. Нажмите "New +" → "Background Worker"
2. Подключите ваш GitHub репозиторий: `codevai-team/codev_bot`
3. Render автоматически обнаружит `render.yaml`

### 3. Добавьте переменные окружения
В настройках сервиса добавьте:

**Обязательные:**
- `BOT_TOKEN` - токен вашего Telegram бота (получите у @BotFather)
- `DATABASE_URL` - URL PostgreSQL базы данных

**Опциональные:**
- `IMGBB_API_KEY` - ключ API для загрузки изображений (получите на imgbb.com)

### 4. Настройте базу данных PostgreSQL
1. В Render создайте новую PostgreSQL базу: "New +" → "PostgreSQL"
2. Скопируйте "External Database URL"
3. Добавьте его как переменную `DATABASE_URL` в настройках бота

### 5. Инициализируйте базу данных
После первого деплоя нужно создать таблицы:
1. Подключитесь к базе через psql или pgAdmin
2. Выполните SQL из файла миграции (если есть)
3. Или запустите скрипт инициализации

### 6. Добавьте себя как администратора
Используйте скрипт `add_me_as_admin.py` или `init_admin.py`

## Автоматический деплой
Render автоматически деплоит при каждом push в main ветку GitHub.

## Логи
Смотрите логи в реальном времени в панели Render: "Logs" → "Live"

## Стоимость
- Background Worker: $7/месяц (есть бесплатный trial)
- PostgreSQL: бесплатно до 1GB

## Альтернатива - Railway.app
Если хотите бесплатный вариант, используйте Railway.app:
1. Зарегистрируйтесь на railway.app
2. "New Project" → "Deploy from GitHub repo"
3. Выберите репозиторий
4. Добавьте переменные окружения
5. Railway автоматически определит Python и запустит бота
